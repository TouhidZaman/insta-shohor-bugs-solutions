JavaScript FAQ Section

JavaScript (JS) is an interpreted language. that means it will not pass through a compiler to translate into bytecode before it run as other compiled language do like C++. JavaScript has no compilation step. So, an interpreter reads the JavaScript code, interprets each line, and runs it. now a days, modern browsers uses Just-In-Time (JIT) compilation, which compiles JavaScript to executable bytecode at run time. So, code optimization can be done while the code is running.

JavaScript is a single-threaded synchronous programming language. That means it runs one line of code at a time manner and there is no possibility of running code in parallel. So, when the asynchronous task appears that is where the event loop comes into picture ! event loop handle the asynchronous task by collecting and processing events, and executing queued sub-tasks. Asynchronous task waits for execution and when the function stack becomes empty, then it gets loaded onto the stack. Event loop takes the first event from the Event Queue and places it onto the stack. Thats how event loop gives us an illusion of being multithreaded even though JS is single-threaded.

Though sessionStorage is similar to localStorage theres some the difference between them. For example, data in localStorage doesn't expire but data in sessionStorage is temporary. A unique page session gets created whenever a document is loaded in a particular tab in the browser. Session Storage is session based and works per window or tab. So, after session ends data gets cleared automacillay. But localStorage is not session-based, it must be deleted via javascript or manually
